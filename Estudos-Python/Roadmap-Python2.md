# Roadmap Estruturado de Estudos – Machine Learning com Base Matemática

> **Carga diária:** ~1 hora
>
> **Frequência:** todos os dias
>
> **Objetivo final:** construir base sólida e consciente em **Machine Learning**, entendendo dados, matemática e modelos — sem superficialidade e sem academicismo excessivo.

---

## Como usar este roadmap (leia antes)

- Este arquivo descreve **o que estudar e por quê**.
- Os **exercícios ficam exclusivamente** no arquivo: `Exercicios_Roadmap_Python_ML.md`.
- Cada dia foi pensado para ~1h, dividida em:

  - 40–45 min de estudo conceitual
  - 15–20 min de revisão, anotações ou leitura reflexiva

- Não avance se o conceito do dia **não estiver claro**.

---

## Visão geral das fases

1. **Fase 1 – Python para ML (nível intermediário)**
   Pensar corretamente em Python para problemas de dados e ML

2. **Fase 2 – Dados e Análise (nível técnico)**
   Extrair sinal, entender dados reais e preparar terreno para modelos

3. **Fase 3 – Matemática Essencial para ML**
   Álgebra linear, probabilidade e estatística aplicadas

4. **Fase 4 – Machine Learning (core)**
   Modelos, validação, generalização e métricas

5. **Fase 5 – Consolidação e Projeto Final**
   Integração completa do aprendizado

## FASE 1 – Python para Machine Learning (Nível Intermediário)

> **Objetivo da fase:** deixar de usar Python apenas como linguagem e passar a usá‑lo como **ferramenta de raciocínio para ML**.

### Semana 1 – Modelo mental correto em Python

**Dia 1 – Tipos, mutabilidade e estado**

- Tipos mutáveis vs imutáveis
- Referência vs cópia
- Por que isso causa bugs silenciosos
- Resultado esperado: entender alterações inesperadas em dados

**Dia 2 – Estruturas de dados em profundidade**

- list, tuple, set, dict
- Casos de uso reais em dados
- Noção intuitiva de custo computacional
- Resultado esperado: escolher a estrutura correta conscientemente

**Dia 3 – Funções como transformações matemáticas**

- Funções puras vs efeitos colaterais
- Entrada → processamento → saída
- Funções como blocos de pipeline
- Resultado esperado: escrever funções previsíveis

**Dia 4 – Expressividade e padrões idiomáticos**

- comprehensions
- enumerate, zip, any, all
- Leitura de código limpo
- Resultado esperado: menos código, mais clareza

**Dia 5 – Organização e modularização**

- Separação de responsabilidades
- Estrutura mínima de projeto
- Resultado esperado: código sustentável

**Dia 6 – Debug consciente**

- Leitura de traceback
- Estratégias reais de depuração
- Resultado esperado: resolver erros com método

**Dia 7 – Revisão estruturada**

- Revisar conceitos da semana
- Conectar Python com problemas de ML
